Simple Character Model Importer - Automated Import of Any Dynamic Model into Gmod (Utility / Tool)

[b]You can download any MMD model from [url=https://www.aplaybox.com/]Aplaybox[/url], [url=https://bowlroll.net/file/index]Bowroll[/url] etc... and instantly import it into Garry's Mod with this addon without restarting your game. [/b] This addon does real-time import of any standard PMX (MikumikuDance) formatted, dynamic (character) models into Garry's Mod.

[h1]Introduction[/h1]
You need to use the provided opensourced importer tool to convert supported model files into Garry's Mod-ready mesh/material data, then spawn them in-game with the Toolgun. Download the tool here: [url=https://github.com/SheepyLord/Gmod-Simple-Character-Model-Importer/releases] GitHub: SheepyLord/Gmod-Simple-Character-Model-Importer
[/url]

Have you noticed that, unlike most modern games, you have to port a character into Source Engine format before you can actually use it in Garry's Mod? That process is slow, repetitive, and difficult, which leaves the community with far fewer characters to utilize. Well, that is no longer the case. Here we introduce "Simple Dynamic Model Importer" to fully address this issue.

This addon is for players and creators who want to bring [b]custom character models[/b] into Garry's Mod without going through the full Source engine model porting and compile pipeline. This addon currently only supports models in standard .pmx (Polygon Model eXtended for MikuMikuDance) format only.

[h2] Adding "Dynamic Model Importer" to the Tools, "Model Importer" Panel [/h2]

[b]As soon as you trust yourself, you will know how to live.[/b]
 --- Johann Wolfgang von Goethe - Faust - AD 1790

[img]https://i.imgur.com/fVVaDCS.gif[/img]

Addon has Chinese (中文), English (English), French (Français), Japanese (日本語), Korean (한국어), and Russian (Русский) translation.

[h1]Features[/h1]
- Import almost any character model as playermodel and NPC into Garry's Mod
- Spawn imported models in-game with the Toolgun or Spawnmenu
- Supports characters with packed PBR textures from Unreal / Unity Engine. (Visible in RTX-Remix only)
- Select which parts of the model should be imported.
- Models are server-sided and shareable between clients
- Export your imported models as Garry's Mod addons and release in workshop
- Save / Load between sessions

[h2]How to use[/h2]
[list=1]
[*]Subscribe to this Workshop item.
[*]Download the open-source Python importer tool from GitHub.
[*]Enter a valid [b]Display Name[/b] and import the model into your Garry's Mod install.
[*]Run the open-source Python importer tool from GitHub.
[*]Launch Garry's Mod.
[*]Open the Toolgun and go to [b]Model Importer -> Dynamic Model Importer[/b].
[*]Refresh the imported model list if needed.
[*]Left-click to place it in the world as NPC, right click as Ragdoll.
[/list]

[h3]Known Issues[/h3]
- Crash or fail on complicated model, especially when importing / loading models with more than 128k vertices.
- Slight offset between the autogenerated physics / hitbox model and the acutal mesh.
- Character Models in other format (.fbx and .uemodels models) must be converted into standard .pmx model for the importer to work. If you character can play dances in Mikumikudance, it will work with this tool.

Это дополнение предназначено для игроков и разработчиков, которые хотят импортировать [b]пользовательские модели персонажей[/b] в Garry's Mod, минуя полный процесс переноса и компиляции моделей в движке Source. В настоящее время это дополнение поддерживает только модели в стандартном формате .pmx (Polygon Model eXtended для MikuMikuDance).

[h2] Добавление «Dynamic Model Importer» в панель «Model Importer» в разделе «Tools» [/h2]

[h1]Особенности[/h1]
- Импорт практически любой модели персонажа в качестве модели игрока или NPC в Garry's Mod
- Появление импортированных моделей в игре с помощью Toolgun или Spawnmenu
- Поддержка персонажей с упакованными PBR-текстурами из движков Unreal / Unity. (Видно только в RTX-Remix)
- Выбор частей модели, которые следует импортировать.
- Модели хранятся на сервере и доступны для всех клиентов
- Экспорт импортированных моделей в виде аддонов для Garry's Mod и публикация в Workshop
- Сохранение / загрузка между сессиями

[h2]Как использовать[/h2]
[list=1]
[*]Подпишитесь на этот предмет в Workshop.
[*]Скачайте инструмент-импортер на Python с открытым исходным кодом с GitHub.
[*]Введите действительное [b]Отображаемое имя[/b] и импортируйте модель в вашу установку Garry's Mod.
[*]Запустите инструмент-импортер на Python с открытым исходным кодом с GitHub.
[*]Запустите Garry's Mod.
[*]Откройте Toolgun и перейдите в [b]Model Importer -> Dynamic Model Importer[/b].
[*]При необходимости обновите список импортированных моделей.
[*]Щелкните левой кнопкой мыши, чтобы разместить модель в мире в качестве NPC, правой кнопкой — в качестве Ragdoll.
[/list]

[h3]Известные проблемы[/h3]
- Сбой или ошибка при работе со сложными моделями, особенно при импорте/загрузке моделей с более чем 128 тыс. вершин.
- Незначительное смещение между автоматически сгенерированной физической моделью / моделью хитбокса и фактической сеткой.
- Модели персонажей в других форматах (модели .fbx и .uemodels) необходимо преобразовать в стандартный формат .pmx, чтобы импортер работал. Если ваш персонаж может танцевать в Mikumikudance, он будет работать с этим инструментом.

[h3]Раскрытие информации об ИИ[/h3]
Эта программа / дополнение содержит некоторые предварительно обученные модели ИИ или алгоритмы. Программы, связанные с ИИ, будут запускаться только локально на вашем компьютере, если пользователь не решит иначе. GPU не требуется, если пользователь явно не включит вычисления на GPU. Для некоторых дополнительных шагов требуется ключ API. Ваша информация не сохраняется и не отправляется разработчику этого инструмента. 

[h3]Невыпущенная версия[/h3]
В связи с нормативными требованиями доступ к сервисам ChatGPT/CodeX или предоставление доступа к ним в некоторых странах и территориях запрещены. Полностью автоматизированная версия с производительностью, сопоставимой с человеческой, на основе DeepSeek будет выпущена после завершения нашей локальной настройки под конкретные задачи и полного устранения зависимости от OpenAI.
